# Product Requirements & Specifications

This directory contains plans, specifications, and design documents for **future or in-progress work**.

## Purpose

Files here should describe:
- What we want to build (requirements)
- How we plan to build it (specifications)
- Why we're building it (vision documents)
- When we plan to build it (roadmaps)

## What Belongs Here

- **PRDs**: Product requirement documents for new features
- **Specifications**: Technical design documents
- **Plans**: Implementation plans and migration guides
- **Roadmaps**: Project timelines and priorities
- **Vision**: Long-term goals and architectural direction

## What Does NOT Belong Here

- Documentation for existing features (use `/docs` directory)
- API references for implemented code (use `/docs` directory)
- User guides and how-tos (use `/docs` directory)

## Naming Convention

Use descriptive names with suffixes indicating document type:
- `*_prd.md` - Product requirement documents
- `*_spec.md` - Technical specifications
- `*_plan.md` - Implementation plans
- `*_roadmap.md` - Timeline and priority documents
