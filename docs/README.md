# Documentation

This directory contains user-facing documentation describing **what exists** in the project.

## Purpose

Files here should document:
- How to use existing features
- API references for implemented code
- Configuration guides
- Troubleshooting and FAQs

## What Belongs Here

- **Guides**: Step-by-step instructions for common tasks
- **References**: API documentation, command lists, configuration options
- **Descriptions**: How existing systems work

## What Does NOT Belong Here

- Feature specifications or PRDs (use `/prd` directory)
- Implementation plans for future work (use `/prd` directory)
- Design documents for unbuilt features (use `/prd` directory)
- Roadmaps or vision documents (use `/prd` directory)
